# ytx

`ytx` is a local-first command-line tool for extracting YouTube transcripts. It prioritizes official subtitles, falls back to local audio transcription using `faster-whisper`, and optionally translates the transcript into a target language using `argostranslate`.

## Features
- **Smart Fallback:** Prefers official YouTube subtitles (avoids auto-generated by default).
- **Local Transcription:** If no official subtitles exist, downloads the audio and transcribes it locally using [faster-whisper](https://github.com/SYSTRAN/faster-whisper).
- **Offline Translation:** Translates transcripts to your target language using [Argos Translate](https://github.com/argosopentech/argos-translate).
- **Multiple Formats:** Output to `srt` (default), `vtt`, `txt`, or `json`.
- **Batch Processing:** Pass a text file with a list of URLs to process them in bulk.

## Requirements
- Python 3.10+
- `ffmpeg` (required for reliable audio extraction)
- `yt-dlp`

### macOS System Dependencies
```bash
brew install ffmpeg yt-dlp
```

## Installation
Clone this repository and install it via pip:
```bash
git clone <your-repo-url>
cd ytx
pip install .
```

You can verify your environment and dependencies by running:
```bash
ytx doctor
```

## Usage
```bash
ytx <youtube-url> [options]
```

### Examples
**1. Download official subtitles (fallback to local transcription if missing):**
```bash
ytx "https://youtube.com/watch?v=VIDEO_ID"
```

**2. Strictly download official subtitles only (fail if none exist):**
```bash
ytx "https://youtube.com/watch?v=VIDEO_ID" --official-only
```

**3. Transcribe and translate to Spanish (`es`):**
```bash
ytx "https://youtube.com/watch?v=VIDEO_ID" --to es
```

**4. Batch process multiple URLs from a file and output as plain text:**
```bash
ytx --input urls.txt --format txt
```

## Options
| Option | Description |
|--------|-------------|
| `--official-only` | Download only official subtitles; error if absent. |
| `--allow-auto-subs` | Permit YouTube auto-generated subtitles as a fallback before transcription. |
| `--source-lang <code>` | Prefer a specific source language code; otherwise auto-detect during transcription. |
| `--to <code>` | Translate transcript to the target language code (e.g., `en`, `es`, `fr`). |
| `--format <format>` | Choose output format: `srt` (default), `vtt`, `txt`, or `json`. |
| `--model <size>` | Select the `faster-whisper` model size (`tiny`, `base`, `small`, `medium`, `large-v3`). Default is `small`. |
| `--output <dir>` | Choose output directory. Default is `output/`. |
| `--keep-audio` | Keep the downloaded audio file after transcription. |
| `--input <file>` | Read URLs from a text file for batch processing. |
| `--verbose` | Show full internal tool output for debugging. |

## Output Structure
By default, `ytx` creates an `output` directory and groups files by video title:
```text
output/
  Video_Title/
    source.info.json
    source.original.en.srt       # If official subtitles were found
    source.transcribed.srt       # If local transcription was used
    source.translated.es.srt     # If translation was requested
```
